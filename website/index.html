Dev Container

Putting a development environment within a container allows me to isolate my dependencies, and not worry about what packages are already installed on my laptop.

A dev container allows me to to experiment with upgrades - if things don't work out, I can quickly undo and go back to a known good state. Without a dev container, downgrading things can be hit and miss

Makefiles

Splitting the content of the Makefile into smaller files, makes each part easier to reason about.
